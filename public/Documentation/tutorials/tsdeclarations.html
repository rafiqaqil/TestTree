Introduction

The typescript/ folder contains samples in typescript that utilize the jscharting declarations file located in the JSC/ folder.

A tsconfig.json file in the typescript folder specifies details to enable some IDEs and the command line compiler to generate the js files. This is required before these samples will run in the browser.

To compile without an IDE, use the command prompt with the following commands:

cd typescript tsc

If editing sample ts code, this command will continually compile changes as you make them:

tsc --watch

A triple dash reference at the top of sample ts files is used to point to the JSCharting declarations file which enables validation of chart configuration properties.

The code expansion feature is not compatible with typescript, therefore only fully expanded property names can be used.

Docs API mapping and usage

Configuration object names such as 'axis' are prefixed with 'JSC', and postfix 'Config' resulting in 'JSCAxisConfig'.

Other objects are only prefixed with JSC.

It is recommended that config objects are marked with types as follows:

let yAxis = {/*y axis properties*/} as JSCAxisConfig;
