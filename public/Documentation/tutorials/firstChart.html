Your First Chart

To begin, create a new HTML file. This file can be created within the JSC bundle or built off of a sample in /Samples. In order to use the charts in any other location, copy the JSC folder from within the bundle to your working directory. This is the minimum requirement to work with JSCharting. First, include the JSCharting library in your HTML file as follows: <script type="text/javascript" src="JSC/JSCharting.js"></script> In the body of the HTML page, add a div element with a specific ID attribute to contain the chart graphics. <div id="chartDiv" style="width: 100%; height: 400px"></div> The chart will automatically conform to the container size by default. Now add a script block to define the chart options to initialize and render the chart in the div#chartDiv tag. The div ID can be specified through the targetElement property in config or as the first argument in the chart constructor. <script type="text/javascript">

var config = {
series: [
{
name: 'Purchases',
points: [
['Jan',29.9],
['Feb',34.6],
['Mar',33.8]
]
}]
};


/*Now initialize the chart with this config setting and a chart will render in place of the div tag with the 'chartDiv' ID.*/ var chart = new JSC.Chart('chartDiv',config);


</script> Open this page in your web browser to view your newly configured chart! It is best to include scripts and script tags at the end of the HTML page. This way scripts will not block rendering of the HTML page. Include CSS files at the beginning.

If jQuery is used on the page, the chart will become a jQuery plugin and can be used to initialize the chart more easily.

//Initialize chart $('#chartDiv').JSC(config); //Get chart reference. var chart = $('#chartDiv').JSC(); Note: It is important to be aware of the chart's Code Expansion feature when looking at sample code. In order to specify the chart types effectively, it is also important to understand Enum Chaining feature. More on using chart types is available in the chart types tutorials.
